## Что нужно установить на чистый компьютер для разработки этой программы

Чтобы написать и запустить приложение (Python + PyQt5 + PostgreSQL), вам потребуется следующее ПО:

### 1. Python (интерпретатор)
- Скачайте и установите **Python 3.10 или новее** с официального сайта: [python.org](https://www.python.org/downloads/)
- **Важно:** при установке на Windows отметьте галочку **«Add Python to PATH»** (добавить Python в переменные окружения).

### 2. PostgreSQL (сервер базы данных)
- Скачайте и установите **PostgreSQL** (последнюю стабильную версию) с [postgresql.org](https://www.postgresql.org/download/)
- В процессе установки:
  - Запомните или задайте **пароль для суперпользователя postgres** (например, `postgres` – используется в настройках по умолчанию).
  - Оставьте порт по умолчанию: `5432`.
  - Установите также **pgAdmin** (графический инструмент для управления БД) – он полезен, но не обязателен.
- После установки убедитесь, что служба PostgreSQL запущена (обычно добавляется в автозагрузку).

### 3. Среда разработки (любая на ваш выбор)
Рекомендуемые бесплатные варианты:
- **VS Code** (легковесный) + расширение Python – [code.visualstudio.com](https://code.visualstudio.com/)
- **PyCharm Community Edition** – [jetbrains.com/pycharm/](https://www.jetbrains.com/pycharm/download/)
- **Простой текстовый редактор** (Notepad++, Sublime Text) – но это менее удобно.

### 4. Необходимые Python-библиотеки
После установки Python откройте **командную строку** (терминал) и выполните:

```bash
pip install PyQt5 psycopg2-binary bcrypt
```

- `PyQt5` – для графического интерфейса (desktop-приложения).
- `psycopg2-binary` – драйвер для подключения к PostgreSQL.
- `bcrypt` – для безопасного хеширования паролей.

### 5. Дополнительно (опционально)
- **Git** – для контроля версий (если будете использовать репозиторий).
- **Docker** – если хотите поднять PostgreSQL в контейнере (вместо локальной установки), но для простоты лучше установить PostgreSQL напрямую.

### 6. Папка с изображениями для приложения
Создайте в папке вашего проекта каталог `images` и положите туда два файла:
- `default.png` – заглушка для товаров без фото (можно любую маленькую картинку).
- `logo.png` – логотип приложения (или просто любой PNG).

## Краткая схема установки (по шагам для Windows)

1. **Скачать Python 3.12** → установить, в первом окне включить «Add Python to PATH».
2. **Скачать PostgreSQL** → установить, ввести пароль `postgres` (или свой), запомнить его.
3. **Перезагрузить компьютер** (чтобы PATH обновился).
4. **Открыть командную строку** (Win+R → cmd).
5. Выполнить `pip install PyQt5 psycopg2-binary bcrypt`.
6. **Создать папку проекта**, например `C:\my_app`.
7. Внутри неё создать папку `images` и положить туда `default.png` и `logo.png`.
8. Скопировать в папку проекта **код приложения** (например, `app_postgres.py`).
9. Запустить приложение: `python app_postgres.py`.

## Что нужно скачать для Windows Forms приложения (C# + PostgreSQL / SQLite)

Если вы хотите написать **такое же приложение (авторизация, товары, корзина, заказы)**, но на **Windows Forms** (C#), то вам потребуется другой набор инструментов, нежели для Python + PyQt. Ниже — список необходимого ПО для чистой системы.

### 1. Среда разработки (обязательно)

**Visual Studio 2022 Community** (бесплатная)  
👉 Скачать: [visualstudio.microsoft.com/ru/downloads/](https://visualstudio.microsoft.com/ru/downloads/)

При установке выберите рабочие нагрузки:
- **.NET desktop development** (разработка классических приложений .NET)
- **Data storage and processing** (для работы с БД — опционально)

Альтернативы (более лёгкие):
- **.NET SDK** + **Visual Studio Code** (с расширением C#) – но для Windows Forms удобнее полноценная Visual Studio.

### 2. .NET (входит в Visual Studio)

Visual Studio установит нужную версию .NET. Для Windows Forms можно использовать:
- **.NET Framework 4.8** (идёт с VS, встроен в Windows 10/11)
- **.NET 6/8** (современный, кроссплатформенный) — предпочтительнее.

### 3. База данных (два варианта)

#### Вариант А: PostgreSQL (серверная БД)
- Скачать и установить **PostgreSQL** (как я описывал ранее) — [postgresql.org](https://www.postgresql.org/download/)
- Драйвер для C#: **Npgsql** – устанавливается через NuGet (встроен в Visual Studio).

#### Вариант Б: SQLite (встраиваемая БД, без установки сервера)
- Ничего дополнительного скачивать не нужно. SQLite поддерживается из коробки через `System.Data.SQLite` (NuGet) или `Microsoft.Data.Sqlite` (NuGet).  
- **Проще для старта** – не требует настройки PostgreSQL.

### 4. Дополнительные библиотеки (через NuGet)

В Visual Studio откройте **Управление пакетами NuGet** и установите:

| Для PostgreSQL | Для SQLite |
|----------------|-------------|
| `Npgsql` | `Microsoft.Data.Sqlite` |
| `BCrypt.Net-Next` (хеширование паролей) | `BCrypt.Net-Next` |

NuGet — это встроенный менеджер пакетов, ничего отдельно скачивать не нужно.

### 5. Инструменты для работы с БД (опционально)

- **pgAdmin** (идёт с PostgreSQL) – для просмотра таблиц.
- **DBeaver** (бесплатный универсальный клиент) – удобен и для PostgreSQL, и для SQLite.

## Краткий план действий для Windows Forms (C# + SQLite)

1. Установить **Visual Studio 2022 Community** (выбрать нагрузку `.NET desktop development`).
2. Создать новый проект: **Windows Forms App (.NET Framework 4.8)** или **Windows Forms App (.NET 6/8)**.
3. Открыть **NuGet** (ПКМ по проекту → Manage NuGet Packages) и установить:
   - `Microsoft.Data.Sqlite`
   - `BCrypt.Net-Next`
4. Написать код (аналогичный логике Python-версии, но на C#).
5. Создать папку `images` в папке `bin\Debug` (или настроить копирование ресурсов).
6. Запустить.

При использовании **PostgreSQL** вместо SQLite: установите `Npgsql` вместо `Microsoft.Data.Sqlite`, настройте строку подключения.
